# 🔧 החלטות טכניות - VacationVibe

## 🏗️ ארכיטקטורה

### למה לא השתמשנו במסמך ארכיטקטורה נפרד?
1. **פשטות המערכת**: המערכת מבוססת על ארכיטקטורת client-server סטנדרטית
2. **תיעוד מובנה**: התיעוד הקיים ב-README וב-project-structure מספק את המידע הנדרש
3. **גמישות**: מאפשר שינויים מהירים ללא צורך בעדכון מסמכים מרובים
4. **תחזוקה**: קל יותר לתחזק מסמך אחד מאשר מספר מסמכים

## 🛠️ בחירות טכנולוגיות

### Frontend
1. **React + TypeScript**
   - טיפוסים חזקים ובטוחים
   - תמיכה קהילתית נרחבת
   - כלי פיתוח מצוינים

2. **Material-UI**
   - עיצוב מודרני ומקצועי
   - תמיכה מלאה ב-RTL
   - רכיבים מוכנים לשימוש
   - תמיכה בתימות

3. **Redux Toolkit**
   - ניהול מצב פשוט ויעיל
   - כלים מובנים לאסינכרוניות
   - DevTools מצוינים

4. **React Router**
   - ניווט חלק ויעיל
   - תמיכה בניווט מקונן
   - שליטה בהרשאות

### Backend
1. **Node.js + Express**
   - ביצועים מעולים
   - קל לפיתוח ותחזוקה
   - אקוסיסטם עשיר

2. **TypeORM**
   - תמיכה ב-TypeScript
   - API אינטואיטיבי
   - ביצועים טובים
   - מיגרציות אוטומטיות

3. **MySQL**
   - אמינות גבוהה
   - ביצועים טובים
   - קל לתחזוקה
   - תמיכה בטרנזקציות

4. **JWT**
   - סטנדרט מקובל לאימות
   - קל ליישום
   - ביצועים טובים
